Project Design and Architecture

Pawtner Hope Foundation Management System

This project follows a 3-layer web application architecture using Go for the backend, HTML/CSS/JavaScript for the frontend, and MongoDB with in-memory runtime structures for storage and fast access.

2. Frontend Design

  • `index.html` provides the home page and public information.
  • `auth.html` handles registration, OTP verification, and login.
  • `service.html` manages service booking and user reviews.
  • `dashboard.html` is the normal user dashboard.
  • `admin.html` is the admin control panel.

3. Backend Design

The backend is implemented in Go using the standard net/http package. It exposes REST-style endpoints, validates requests, processes business logic, and returns JSON responses to the frontend.

4. Middleware Layer

  • recoverPanic() catches runtime panics and returns safe JSON errors.
  • enableCORS() adds CORS headers and handles preflight requests.
  • requireAdmin() is used for admin-only route protection.
  • ValidateToken() validates JWT-based authentication tokens.

5. Business Modules

  • Pet management
  • User registration and OTP verification
  • Login with failed-attempt lockout
  • Service booking management
  • Admin booking review
  • User review submission
  • Donation processing and receipt handling
  • Adoption inquiries and contact messages
  • Statistics and admin dashboard summaries

6. Data Layer

The application uses both in-memory data structures and MongoDB. In-memory slices and maps provide quick runtime access, while MongoDB provides permanent persistence for pets, users, bookings, reviews, donations, inquiries, and contact messages.

7. Worker Pool / Concurrency Design

Go concurrency is used through goroutines and channels. The project now includes an email worker pool, along with separate background workers for payment processing and payment confirmation handling.

8. Request Flow Example

Service Booking Flow

User -> service.html form
     -> POST /api/bookings
     -> Go handler validates input
     -> booking saved in memory + MongoDB
     -> notification sent to worker pool
     -> response returned to UI

User Review Flow

User -> service.html review form
     -> POST /api/reviews
     -> backend validates booking and email
     -> review saved in MongoDB
     -> rating/statistics updated
     -> admin can view review in admin panel

9. Admin Architecture

The admin dashboard allows admins to manage pets, adoption inquiries, bookings, donations, messages, and user-submitted reviews. Admin access is controlled through JWT validation and admin-only authorization checks.

10. Conclusion

This project uses a modular and scalable full-stack design with a frontend presentation layer, Go backend application layer, MongoDB persistence layer, and asynchronous worker pool for background processing. This makes the system suitable for pet adoption, service booking, and foundation management operations.
